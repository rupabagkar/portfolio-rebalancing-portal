## Setup

1. Install dependencies:
```bash
npm install
```

2. Configure environment:
```bash
cp .env.example .env
# Edit .env with your configuration
```

3. Setup database:

### Option A: Raw SQL Migrations
```bash
# Apply migrations
npm run db:migrate

# Run seed data
npm run db:seed

# Reset database (migrate + seed)
npm run db:reset
```

### Option B: Prisma (Recommended)
```bash
# Generate Prisma client
npm run prisma:generate

# Run migrations
npm run prisma:migrate

# Open Prisma Studio (visual database browser)
npm run prisma:studio
```

4. Start server:
```bash
npm run dev
```

## Database Schema


### Tables Generated from API Contracts

The database schema is **automatically inferred** from your API endpoints with intelligent field type mapping:

- `*Id` fields → UUID/CHAR(36) with foreign key relationships
- `email` → VARCHAR(255) with UNIQUE constraint
- `*amount/*price/*balance` → DECIMAL(15,2) for monetary precision
- `*date/*time/*At` → TIMESTAMP fields
- `status` → VARCHAR with CHECK constraints (enums)
- Arrays/Objects → JSONB (PostgreSQL) or JSON (MySQL)

### Migration Files

- **Location**: `database/migrations/*.sql`
- **Seed Data**: `database/seeds/seed.sql`
- **Prisma Schema**: `prisma/schema.prisma`

### Prisma Commands

```bash
npx prisma generate     # Generate Prisma client
npx prisma migrate dev  # Create and apply migration
npx prisma studio       # Visual database browser
npx prisma db push      # Push schema without migration
```
